## 2. 제품 개요

스마트 점검 리포트는 임대 주택의 시설 상태를 시공업자가 점검하고, 점검 결과를 임대인과 임차인이 같은 기준으로 확인할 수 있도록 리포트화하는 웹 서비스이다.

시공업자는 점검 유형에 따라 **공간별 전체 점검** 또는 **문제 항목 점검**을 작성한다. AI 점검 도우미는 시공업자에게 어떤 사진을 촬영해야 하는지, 어떤 자료를 확인해야 하는지, 어떤 표현을 피해야 하는지, 전문 의견을 어떻게 작성하면 좋은지를 JSON 형태로 안내한다.

생성된 리포트는 임대인과 임차인에게 즉시 공개되며, PDF 저장과 공유 링크 접근이 가능하다. 생성 완료 리포트는 증빙 성격을 가지므로 수정/삭제하지 않고, 수정이 필요한 경우 새 리포트를 생성한다.

## 3. 제품 목표와 비목표

### 3.1 제품 목표

| 목표 | 설명 |
|---|---|
| 점검 기록 표준화 | 시공업자가 시설 상태를 일관된 형식으로 기록 |
| 임대인 관리 편의성 향상 | 원격 임대인이 호실 상태를 리포트로 확인 |
| 임차인과 상태 공유 | 임차인이 본인 호실의 점검 결과를 동일한 기준으로 확인 |
| 분쟁 예방 보조 | 입주 전, 정기, 퇴거, 긴급, 수리 점검 이력을 누적하여 향후 분쟁 가능성 감소 |
| AI 기록 보조 | 법적 판단이 아닌 사진·자료·표현·전문 의견 작성 보조 |
| 발표 시연 완성도 확보 | 실제 DB, API, AI 연동, 리포트 출력, 비교 기능까지 동작 |

### 3.2 비목표

| 비목표 | 설명 |
|---|---|
| 법적 책임 판단 | 임차인 책임, 임대인 책임, 고의·과실 판단 제공하지 않음 |
| 판례 기반 법률 자문 | 법률 판단이나 소송 대응 문서 작성 제외 |
| AI 이미지 판독 | 이미지를 AI에게 직접 전달해 하자 판정하지 않음 |
| 보증금 공제 산정 | 비용 산정, 견적, 공제 판단 제외 |
| 생성 완료 리포트 수정 | 생성 완료 후 원본 리포트 직접 수정 불가 |
| 사용자 인증 시스템 | 1차 구현에서는 로그인/회원가입 제외, 데모 사용자 선택으로 대체 |

## 6. 사용자 역할과 권한

### 6.1 역할

| 역할 | 기능 |
|---|---|
| 시공업자 | 점검 작성, 사진 첨부, AI 점검 도우미 사용, 점검 제출, 리포트 자동 생성 |
| 임대인 | 보유 호실 리포트 확인, 리포트 비교, 확인 완료, PDF 저장, 공유 링크 생성 |
| 임차인 | 본인 호실 리포트 확인, 확인 완료, PDF 저장, 공유 링크 생성 |
| 공유 링크 접근자 | 공유 URL로 리포트 조회, PDF 저장 |

### 6.2 데모 사용자 선택

로그인/회원가입은 구현하지 않는다. 첫 화면에서 데모 사용자를 선택하고, 선택값은 `localStorage`에 저장한다.

| 사용자 | 역할 | 접근 범위 |
|---|---|---|
| 이시공 | 시공업자 | 본인이 작성한 점검 수정/삭제 가능 |
| 김임대 | 임대인 | 본인 소유 호실의 생성 완료 리포트 접근 가능 |
| 박임차 | 임차인 | 본인 거주 호실의 생성 완료 리포트 접근 가능 |

```text
역할/사용자 선택
→ selectedUserId, selectedRole을 localStorage에 저장
→ API 요청 시 userId 전달
→ 백엔드에서 DB 기준 접근 가능 여부 확인
```

## 7. 핵심 사용자 시나리오

### 7.1 시공업자 - 입주 전 전체 점검

```text
시공업자 홈
→ 새 점검 시작
→ 입주 전 점검 선택
→ 점검 대상 입력
→ 공간별 전체 점검
→ 정상/주의/수리 필요 선택
→ 주의 또는 수리 필요 항목 상세 기록
→ AI 점검 도우미 확인
→ 필요한 사진 첨부
→ 시공업자 최종 의견 작성
→ 점검 제출
→ 리포트 자동 생성
→ 임대인/임차인에게 공개
```

### 7.2 시공업자 - 긴급 누수 문제 항목 점검

```text
시공업자 홈
→ 새 점검 시작
→ 긴급 점검 선택
→ 점검 대상 입력
→ 설비·배관 / 누수 선택
→ 위치와 증상 설명 입력
→ 현장 확인 항목 선택
→ AI 점검 도우미 호출
→ 촬영/확인/표현 주의 카드 확인
→ 사진 첨부
→ 전문 의견 작성
→ 점검 제출
→ 리포트 자동 생성
```

### 7.3 임대인

```text
임대인 선택
→ 통합 리포트 목록
→ 호실/점검 유형/확인 상태 필터
→ 리포트 상세 확인
→ PDF 저장 또는 공유 링크 생성
→ 같은 호실 + 같은 점검 유형 리포트 2개 비교
→ 확인 완료
```

### 7.4 임차인

```text
임차인 선택
→ 본인 호실 리포트 목록
→ 리포트 상세 확인
→ 사진 및 시공업자 의견 확인
→ PDF 저장 또는 공유 링크 생성
→ 확인 완료
```

### 8.1 전체 점검 공간 목록

| 공간 | 기본 항목 |
|---|---|
| 현관 | 현관문, 도어락, 문틀, 신발장 |
| 거실 | 벽지, 바닥, 창호, 콘센트 |
| 주방 | 싱크대, 수전, 배수, 후드 |
| 방1 | 벽지, 바닥, 창호, 콘센트 |
| 방2 | 벽지, 바닥, 창호, 콘센트 |
| 화장실 | 세면대, 변기, 수전, 배수 |
| 베란다 | 창호, 바닥, 배수, 누수 흔적 |
| 보일러룸 | 보일러, 배관, 밸브, 누수 흔적 |

### 8.2 전체 점검 상태값

| 상태 | 점수 | 의미 |
|---|---:|---|
| 정상 | 0 | 특이사항 없음 |
| 주의 | 1 | 기록 필요, 즉시 수리 확정은 아님 |
| 수리 필요 | 2 | 보수, 교체, 추가 조치 검토 필요 |

### 8.3 문제 항목 점검 분야

| 분야 | 대표 문제 항목 |
|---|---|
| 인테리어·마감 | 벽지 찢어짐, 벽지 오염, 바닥 찍힘, 몰딩 파손, 타일 균열 |
| 샷시·창호 | 방충망 찢어짐, 창문 작동 불량, 유리 파손, 실리콘 들뜸, 잠금장치 불량 |
| 설비·배관 | 누수, 배수 불량, 수전 흔들림, 보일러 이상, 배관 부식 |
| 전기 | 콘센트 불량, 조명 미작동, 스위치 불량, 차단기 이상, 배선 노출 |
| 소방·안전 | 감지기 미작동, 소화기 미비, 가스누설 의심, 비상등 불량, 안전 위험 요소 |

### 8.4 현장 확인 항목 선택값

| 선택값 | 의미 |
|---|---|
| 있음 | 현장에서 확인됨 |
| 없음 | 현장에서 확인되지 않음 |
| 확인 필요 | 현재 정보만으로 확인 어려움 |

## 9. 등급 산출 정책

등급은 AI가 아니라 규칙 기반으로 산출한다.

### 9.1 전체 점검 등급

| 등급 | 조건 |
|---|---|
| A | 모든 항목 정상 |
| B | 주의 1~2개, 수리 필요 없음 |
| C | 주의 3개 이상 또는 수리 필요 1개 |
| D | 수리 필요 2개 이상 |
| E | 소방·안전 관련 수리 필요 또는 긴급 조치 필요 |

### 9.2 문제 항목 점검 등급

| 등급 | 조건 |
|---|---|
| B | 경미한 기록 필요 |
| C | 추가 확인 필요 |
| D | 수리 필요 |
| E | 긴급 조치 필요 또는 안전 위험 |

### 9.3 소방·안전 특례

소방·안전 분야에서 `수리 필요` 또는 `긴급 조치 필요`가 선택되면 자동으로 E등급 처리한다.


## 10. AI 점검 도우미

### 10.1 목적

AI 점검 도우미는 시공업자에게 다음 내용을 안내한다.

| 제공 내용 | 설명 |
|---|---|
| 사진 촬영 기준 | 어떤 사진을 찍어야 하는지 안내 |
| 추가 확인 질문 | 어떤 내용을 더 확인해야 하는지 안내 |
| 필요 자료 | 입주 전 리포트, 이전 수리 내역, 신고 기록 등 안내 |
| 표현 주의 | 법적 책임을 단정하지 않도록 문장 주의 |
| 전문 의견 초안 | 시공업자가 수정 가능한 중립적 의견 초안 |

### 10.2 제외 사항

| 제외 항목 | 설명 |
|---|---|
| 이미지 직접 판독 | 사진을 AI에 전달해 하자를 판정하지 않음 |
| 법적 책임 판단 | 임차인 책임, 임대인 책임 판단하지 않음 |
| 고의·과실 판단 | 임차인 고의, 과실 확정 표현 금지 |
| 보증금 공제 판단 | 비용 부담, 공제 가능 여부 판단하지 않음 |
| 판례 자동 검색 | 1차 구현 범위에서 제외 |
| 소송 대응 문서 | 법률 서비스로 오해될 수 있어 제외 |

### 10.3 AI 호출 방식

1차 구현에서는 AI 점검 도우미 호출 시 백엔드가 두 개의 시스템 프롬프트 내용을 함께 사용한다.

| 파일 | 역할 |
|---|---|
| `inspection-guide.system.md` | 사진 기준, 추가 확인 질문, 필요 자료, 표현 주의 생성 |
| `opinion-draft.system.md` | 시공업자 전문 의견 초안 생성 |

하나의 API 호출에서 위 목적을 모두 처리하고, 응답은 JSON으로 반환한다.

### 10.4 AI 응답 JSON 구조

```json
{
  "summary": "현재 점검 상황 요약",
  "actionCards": [
    {

```

## 11. AI 시스템 프롬프트

### 11.1 `backend/prompts/inspection-guide.system.md`

```md
# 역할

너는 임대 주택 시설 점검을 보조하는 AI 점검 도우미이다.

너의 역할은 시공업자가 현장에서 점검 내용을 빠짐없이 기록하도록 돕는 것이다.

너는 법률가가 아니며, 법적 책임 소재를 판단하지 않는다.
너는 이미지를 직접 분석하지 않는다.
너는 보증금 공제, 비용 부담, 임차인 고의, 임대인 책임을 단정하지 않는다.

# 목표

사용자가 입력한 점검 유형, 점검 흐름, 공간 또는 분야, 문제 항목, 위치, 증상 설명, 현장 확인 항목을 바탕으로 다음 내용을 제공한다.

1. 현재 상황 요약
2. 추가로 촬영해야 할 사진
```

```md
# 역할

너는 시공업자가 작성할 점검 전문 의견 초안을 작성하는 AI 보조자이다.

너의 역할은 시공업자가 현장에서 확인한 내용을 바탕으로 중립적이고 전문적인 문장 초안을 제공하는 것이다.

너는 법적 책임을 판단하지 않는다.
너는 비용 부담 주체를 판단하지 않는다.
너는 임차인 고의 또는 임대인 책임을 단정하지 않는다.
너는 이미지 내용을 직접 판독하지 않는다.

# 작성 원칙

- 한국어로 작성한다.
- 2~4문장 정도로 간결하게 작성한다.
- 현장에서 확인된 사실을 먼저 쓴다.
- 이후 추가 확인이 필요한 사항을 쓴다.
- 마지막으로 현재 상태 기준의 조치 권장 사항을 쓴다.
- 법적 책임 또는 비용 부담을 단정하지 않는다.
- "추가 확인이 필요합니다", "이전 기록과 비교가 필요합니다", "수리 이력 확인이 필요합니다", "전문 설비 점검이 필요합니다" 같은 중립 표현을 사용한다.

# 금지 표현

다음 표현을 사용하지 않는다.

- 임차인 책임입니다
- 임대인 책임입니다
- 임차인 고의입니다
- 과실이 확정됩니다
- 보증금에서 공제 가능합니다
- 소송에서 유리합니다
- 판례상 확정됩니다

# 출력 형식
```

## 13. 리포트 정책

### 13.1 리포트 생성

점검 제출 시 리포트가 자동 생성된다.

```text
점검 작성 완료
→ 점검 제출
→ 리포트 자동 생성
→ report snapshot JSON 저장
→ 임대인/임차인에게 즉시 공개
```

### 13.2 Inspection과 Report 분리

| 구분 | 설명 |
|---|---|
| Inspection | 작성 중/제출 전 점검 데이터 |
| Report | 생성 완료된 고정 스냅샷 데이터 |

### 13.3 Report Snapshot

리포트 생성 시점의 데이터를 JSON으로 저장한다.

Report Snapshot에는 다음 정보가 포함된다.

| 데이터 | 설명 |
|---|---|
| 점검 기본 정보 | 점검 유형, 점검일, 생성일 |
| 점검 대상 정보 | 건물명, 주소, 호실 |
| 참여자 정보 | 시공업자, 임대인, 임차인 |
| 등급 정보 | 규칙 기반 산출 등급 |
| 점검 항목 | 전체 점검 항목 또는 문제 항목 |
| 현장 확인 항목 | 문제 항목 점검 시 포함 |
| 사진 정보 | Base64 이미지 및 설명 |
| AI 가이드 | 선택적으로 내부 기록용 저장 |
| 최종 의견 | 시공업자 최종 의견 |
| 주의 문구 | 법적 책임 판단이 아님을 명시 |

### 13.4 리포트 수정/삭제 정책

| 상태 | 수정 | 삭제 |
|---|---|---|
| 작성 중 | 가능 | 가능 |
| 제출 완료 | 가능 | 가능 |
| 리포트 생성 완료 | 불가 | 불가 |

생성 완료 리포트에 수정 요청이 발생하면 기존 리포트를 수정하지 않고 새 리포트를 생성한다.
