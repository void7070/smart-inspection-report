# 스마트 점검 리포트 개발 문서

> 대상 프로젝트: 스마트 점검 리포트  
> 개발 방식: OMC + Claude Code 세션 분할 개발  
> 기술 스택: Vue 3 + Vite + Tailwind CSS / Node.js + Express / SQLite + better-sqlite3 / GPT API  
> 목적: 발표용 데모이지만 DB, API, AI, 리포트 생성, PDF, 공유 링크, 비교 기능까지 실제 동작하는 수준으로 구현

# 프론트엔드 PRD - 세션별 개발 문서

## 1. 프론트엔드 목표

Vue 3 + Vite + Tailwind CSS를 사용하여 스마트 점검 리포트의 역할별 화면과 점검 작성 플로우를 구현한다.

프론트엔드는 API 서버와 통신하여 실제 SQLite 데이터를 조회/저장하고, GPT API 결과를 백엔드를 통해 받아 화면에 JSON 카드 형태로 표시한다.

## 2. 기술 기준

| 항목 | 내용 |
|---|---|
| Framework | Vue 3 |
| Build Tool | Vite |
| CSS | Tailwind CSS |
| Router | Vue Router |
| 상태 관리 | 우선 Vue ref/reactive 또는 Pinia 중 프로젝트 상황에 맞게 결정 |
| API 통신 | fetch 또는 axios |
| 인증 | 로그인 없음, localStorage 기반 selectedUser |
| PDF | `/reports/:id/print` 또는 인쇄용 템플릿에서 `window.print()` |
| 이미지 | 파일 선택 → Base64 변환 → API 전송 |

## 5. F01 - 프론트 기본 세팅

### 목표

Vue 3 + Vite + Tailwind CSS 프로젝트를 구성하고 기본 라우팅과 레이아웃을 만든다.

### 작업 범위

- `frontend/` 프로젝트 생성
- Tailwind CSS 설정
- Vue Router 설정
- 공통 레이아웃 구성
- API client 기본 함수 작성
- 환경변수 `VITE_API_BASE_URL` 사용

### 완료 기준

- `npm install`, `npm run dev` 정상 실행
- `/` 접속 가능
- Tailwind 스타일 적용 확인
- 라우터 기반 페이지 이동 가능

## 6. F02 - 데모 사용자 선택 및 시공업자 시작 화면

### 목표

로그인 대신 데모 사용자를 선택하고 역할별 화면으로 이동한다.

### 작업 범위

- `GET /api/demo/users` 호출
- 시공업자/임대인/임차인 사용자 카드 표시
- 선택 시 `localStorage`에 `selectedUserId`, `selectedRole` 저장
- 역할별 메인 화면 이동
- 시공업자 홈 구현
- 작성 중 점검, 제출 대기 점검 카드 표시

### 완료 기준

- 사용자 선택 후 새로고침해도 선택 상태 유지
- 시공업자 선택 시 `/contractor` 이동
- API 데이터 기반으로 작성 중/제출 대기 점검 표시

## 7. F03 - 전체 점검 UI

### 목표

입주 전/정기점검의 공간별 전체 점검 UI를 구현한다.

### 확정 공간

| 공간 | 항목 |
|---|---|
| 현관 | 현관문, 도어락, 문틀, 신발장 |
| 거실 | 벽지, 바닥, 창호, 콘센트 |
| 주방 | 싱크대, 수전, 배수, 후드 |
| 방1 | 벽지, 바닥, 창호, 콘센트 |
| 방2 | 벽지, 바닥, 창호, 콘센트 |
| 화장실 | 세면대, 변기, 수전, 배수 |
| 베란다 | 창호, 바닥, 배수, 누수 흔적 |
| 보일러룸 | 보일러, 배관, 밸브, 누수 흔적 |

### 상태값

- 정상
- 주의
- 수리 필요

### 작업 범위

- 점검 유형 선택
- 점검 대상 입력
- 전체 점검 공간별 상태값 입력
- 주의/수리 필요 항목만 상세 입력 화면 제공
- 상세 항목 위치, 설명 입력
- 저장 API 연동

### 완료 기준

- 입주 전 점검과 정기 점검은 전체 점검 UI로 진입
- 모든 공간 상태값 입력 가능
- 주의/수리 필요 항목은 상세 입력 가능
- 저장 후 다시 열어도 데이터 유지

## 8. F04 - 문제 항목 점검 UI

### 목표

퇴거 전/후, 긴급, 수리 전/후 점검의 문제 항목 점검 UI를 구현한다.

### 문제 분야

| 분야 | 대표 문제 항목 |
|---|---|
| 인테리어·마감 | 벽지 찢어짐, 벽지 오염, 바닥 찍힘, 몰딩 파손, 타일 균열 |
| 샷시·창호 | 방충망 찢어짐, 창문 작동 불량, 유리 파손, 실리콘 들뜸, 잠금장치 불량 |
| 설비·배관 | 누수, 배수 불량, 수전 흔들림, 보일러 이상, 배관 부식 |
| 전기 | 콘센트 불량, 조명 미작동, 스위치 불량, 차단기 이상, 배선 노출 |
| 소방·안전 | 감지기 미작동, 소화기 미비, 가스누설 의심, 비상등 불량, 안전 위험 요소 |

### 작업 범위

- 점검 유형에 따라 문제 항목 점검 진입
- 분야/문제 항목 선택
- 위치 입력
- 증상 설명 입력
- 현장 확인 항목 선택
- 있음/없음/확인 필요 선택 UI
- 저장 API 연동

### 완료 기준

- 긴급 점검 선택 시 문제 항목 점검 UI 진입
- 현장 확인 항목 선택값 저장 가능
- 선택값은 AI 요청 입력값으로 전달 가능

## 9. F05 - AI 점검 도우미 및 최종 의견

### 목표

AI API 응답을 카드 UI로 렌더링하고, 전문 의견 초안을 최종 의견 편집 영역에 반영한다.

### 작업 범위

- `POST /api/ai/inspection-guide` 호출
- JSON 응답 파싱
- `summary` 표시
- `actionCards` 표시
- `requiredDocuments` 표시
- `cautionPhrases` 표시
- `opinionDraft` 표시
- 초안 적용 버튼
- 시공업자 최종 의견 수정/저장

### 완료 기준

- AI 응답 실패 시에도 수동 의견 작성 가능
- AI 초안은 최종 의견으로 복사 가능
- 최종 의견은 시공업자가 수정 가능
- 리포트에는 최종 의견만 표시

## 10. F06 - 이미지 첨부

### 목표

Base64 이미지 첨부 UI를 구현한다.

### 정책

| 항목 | 제한 |
|---|---|
| 이미지 1장 | 10MB 이내 |
| 항목당 이미지 | 최대 5장 |
| 리포트 전체 이미지 | 최대 20장 |

### 작업 범위

- 이미지 파일 선택
- Base64 변환
- 용량 검증
- 미리보기
- 사진 유형 선택
- 사진 설명 입력
- 이미지 삭제
- API 저장

### 완료 기준

- 10MB 초과 시 오류 표시
- 항목당 5장 초과 방지
- 저장 후 리포트에서 표시 가능

## 12. F08 - 리포트 비교, 공유 링크, PDF

### 목표

리포트 비교, 공유 링크 조회, 인쇄용 리포트 템플릿을 구현한다.

### 비교 조건

- 같은 호실
- 같은 점검 유형
- 정확히 2개
- 수리 전/후 점검은 예외적으로 비교 가능

### PDF

- HTML 리포트 템플릿
- `window.print()`
- `@media print`
- 버튼/내비게이션/입력폼 제외

### 완료 기준

- 비교 조건 불일치 시 오류 표시
- 공유 링크 URL로 리포트 조회 가능
- 인쇄하기 버튼 클릭 시 브라우저 인쇄창 호출
