> 대상 프로젝트: 스마트 점검 리포트

# OMC 개발 세션 분할 계획

## 1. 전체 개발 원칙

본 프로젝트는 OMC와 Claude Code를 활용해 여러 세션으로 나누어 개발한다.  
각 세션은 반드시 자신의 담당 범위만 구현하고, 다음 세션이 이어받을 수 있도록 변경 내역과 검증 결과를 남긴다.

## 2. 최종 구현 목표

| 구분 | 목표 |
|---|---|
| 프론트엔드 | Vue 3 + Vite + Tailwind 기반 역할별 화면 구현 |
| 백엔드 | Node.js + Express + SQLite API 서버 구현 |
| DB | better-sqlite3 기반 schema, seed, CRUD 구현 |
| AI | GPT API 호출, 시스템 프롬프트 2개 사용, JSON 응답 |
| 이미지 | Base64 이미지 SQLite 저장 |
| 리포트 | 점검 제출 시 Report Snapshot JSON 자동 생성 |
| PDF | HTML 리포트 템플릿 + window.print() |
| 공유 링크 | 만료 없는 token 기반 공유 링크 |
| 비교 | 같은 호실 + 같은 점검 유형 리포트 2개 비교, 수리 전/후 예외 허용 |

## 4. 세션별 작업 원칙

### 4.1 공통 규칙

1. 한 세션은 담당 범위 외 파일을 불필요하게 수정하지 않는다.
2. API 계약이 바뀌면 반드시 문서와 프론트/백엔드 양쪽에 반영한다.
3. 생성 완료 리포트는 직접 수정/삭제하지 않는다.
4. AI는 법적 책임 판단을 하지 않는다.
5. 이미지는 AI로 판독하지 않고 Base64로 저장만 한다.
6. 점검 제출 시 리포트는 자동 생성된다.
7. 리포트 생성 시점의 데이터는 Report Snapshot JSON으로 고정 저장한다.
8. 임대인/임차인/시공업자는 모두 공유 링크를 생성할 수 있다.
9. 공유 링크는 만료 시간이 없다.
10. PDF는 서버 생성이 아니라 window.print() 기반이다.
