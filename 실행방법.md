# 실행 방법 (Quick Start)

스마트 점검 리포트 데모 실행 명령어 정리. **Windows / PowerShell 기준.**

- 요구사항: **Node.js 20+** (권장 24)
- 백엔드 포트 **3000**, 프론트엔드 포트 **5173**
- 터미널 **2개** 필요 (백엔드 / 프론트엔드 각각 실행)

---

## 0) 이미 세팅한 적 있으면 — `npm run dev`만

`npm install` · `.env` 생성 · `npm run db:init`을 이미 한 번 했다면
(즉 `backend/node_modules`, `backend/.env`, `backend/app.db`, `frontend/node_modules`가 있으면)
아래 두 줄만 각 터미널에서 실행하면 된다.

```powershell
# 터미널 ① 백엔드
cd backend ; npm run dev      # http://localhost:3000

# 터미널 ② 프론트엔드
cd frontend ; npm run dev     # http://localhost:5173
```

→ 브라우저에서 **http://localhost:5173** 접속.

- 포트 3000이 안 뜨면 죽은 node 프로세스 정리: `Stop-Process -Name node -Force`
- 시연 직전 데모 데이터를 초기화하고 싶을 때만 `npm run db:init` 한 번 더 (평소 재실행엔 불필요)

최초 세팅이거나 위 파일들이 없으면 아래 1)·2)부터 진행.

---

## 1) 백엔드 실행 (터미널 ①)

```powershell
cd backend
npm install
Copy-Item .env.example .env     # 최초 1회만
npm run db:init                 # SQLite 스키마 + 데모 seed 생성 (app.db)
npm run dev                     # http://localhost:3000  (node --watch)
```

확인:

```powershell
curl http://localhost:3000/api/health
# → {"success":true,...}
```

> ⚠️ 포트 3000을 잡고 있는 죽은 node 프로세스가 있으면 옛 데이터가 뜬다.
> 서버 다시 켜기 전에: `Stop-Process -Name node -Force`

---

## 2) 프론트엔드 실행 (터미널 ②)

```powershell
cd frontend
npm install
npm run dev                     # http://localhost:5173
```

브라우저에서 **http://localhost:5173** 접속 (백엔드가 함께 떠 있어야 함).

---

## 3) (선택) AI 실제 사용

```powershell
# backend/.env 의 OPENAI_API_KEY 채운 뒤 백엔드만 재시작
```

키가 없어도 AI 도우미는 기본 템플릿으로 동작하며, 점검 제출은 막히지 않는다.

## 전체 스크립트 참고

### backend (`backend/package.json`)

| 명령어 | 설명 |
|---|---|
| `npm run dev` | 개발 서버 (node --watch) |
| `npm start` | 일반 서버 실행 |
| `npm run db:init` | DB 스키마 + 데모 seed 생성 |
| `npm test` | Vitest 1회 실행 |
| `npm run test:watch` | Vitest watch 모드 |

### frontend (`frontend/package.json`)

| 명령어 | 설명 |
|---|---|
| `npm run dev` | Vite 개발 서버 |
| `npm run build` | 프로덕션 빌드 |
| `npm run preview` | 빌드 결과 미리보기 |
| `npm test` | Vitest 1회 실행 |
| `npm run test:watch` | Vitest watch 모드 |
