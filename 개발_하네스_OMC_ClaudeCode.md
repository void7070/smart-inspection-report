# 스마트 점검 리포트 개발 문서

> 대상 프로젝트: 스마트 점검 리포트  
> 개발 방식: OMC + Claude Code 세션 분할 개발  
> 기술 스택: Vue 3 + Vite + Tailwind CSS / Node.js + Express / SQLite + better-sqlite3 / GPT API  
> 목적: 발표용 데모이지만 DB, API, AI, 리포트 생성, PDF, 공유 링크, 비교 기능까지 실제 동작하는 수준으로 구현

---

# 개발 하네스 - OMC / Claude Code 세션용

## 1. 목적

이 하네스는 OMC와 Claude Code를 사용하여 스마트 점검 리포트를 세션 단위로 안정적으로 구현하기 위한 실행 기준 문서이다.

각 세션은 다음 원칙을 따라야 한다.

1. 세션별 담당 범위만 구현한다.
2. 이전 세션 산출물을 먼저 확인한다.
3. 구현 후 반드시 실행 및 검증 명령을 남긴다.
4. API 계약과 DB schema 변경은 문서에 반영한다.
5. 다음 세션이 이어받을 수 있도록 종료 보고를 작성한다.

---

## 2. 공통 컨텍스트 패킷

새로운 Claude Code 세션을 시작할 때 반드시 아래 내용을 컨텍스트로 제공한다.

```md
# 프로젝트 컨텍스트

프로젝트명: 스마트 점검 리포트

기술 스택:
- frontend: Vue 3 + Vite + Tailwind CSS
- backend: Node.js + Express
- DB: SQLite + better-sqlite3
- AI: GPT API
- 이미지: Base64를 SQLite에 저장
- PDF: HTML 템플릿 + window.print()

핵심 정책:
- 로그인 없음, 데모 사용자 선택 방식
- selectedUserId, selectedRole을 localStorage에 저장
- API 요청 시 userId 전달
- 백엔드에서 DB 기준 권한 확인
- 점검 제출 시 리포트 자동 생성
- Report는 생성 시점 snapshot JSON으로 저장
- 생성 완료 리포트는 수정/삭제 불가
- 수정 필요 시 새 리포트 생성
- AI는 법적 판단/이미지 판독/보증금 공제 판단을 하지 않음
- AI 응답은 JSON
- 공유 링크는 시공업자/임대인/임차인 모두 생성 가능
- 공유 링크 만료 없음
- 공유 링크 화면에서 이름 마스킹하지 않음
- 리포트 비교는 같은 호실 + 같은 점검 유형, 수리 전/후 예외 허용
```

## 4. 공통 검증 명령

### 4.1 백엔드

```bash
cd backend
npm install
npm run dev
```

확인:

```bash
curl http://localhost:3000/api/health
curl http://localhost:3000/api/demo/users
```

### 4.2 프론트엔드

```bash
cd frontend
npm install
npm run dev
```

확인:

```text
http://localhost:5173
```

## 7. 오류 처리 기준

| 상황 | 처리 |
|---|---|
| 권한 없음 | 403 |
| 리소스 없음 | 404 |
| 필수값 누락 | 400 |
| 생성 완료 리포트 수정 시도 | 400 또는 403 |
| 이미지 10MB 초과 | 400 또는 프론트 오류 |
| 항목당 5장 초과 | 오류 표시 |
| 리포트당 20장 초과 | 오류 표시 |
| AI 호출 실패 | 경고 표시, 수동 작성 가능 |
| AI JSON 파싱 실패 | 기본 템플릿 또는 오류 메시지 |
| 비교 조건 불일치 | 400 및 안내 메시지 |
| 공유 토큰 없음 | 404 |

## 8. 세션별 완료 정의

각 세션은 다음을 만족해야 완료된다.

1. 담당 기능이 동작한다.
2. 실행 방법이 문서화되어 있다.
3. 최소 1개 이상의 검증 결과가 있다.
4. 다음 세션이 이어받을 주의사항이 있다.
5. 실패하거나 보류한 기능은 명확히 기록한다.
